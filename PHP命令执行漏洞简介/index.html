简单总结并验证命令执行漏洞

PHP命令执行漏洞

01 命令执行简介

  1. 命令执行的原理及成因
    程序员使用脚本语言（如PHP）进行开发时，需要调用一些外部程序（系统命令或可执行文件），在调用这些命令时，如果将用户的输入作为系统命令的参数拼接到命令行中，并且没有对用户的输入进行过滤，就会造成命令执行漏洞。
    可以总结为两点：
    • 用户输入拼接到系统命令中执行
    • 用户输入未经过过滤(命令不再执行)或净化(去除敏感字,命令继续执行)
  2. 漏洞危害
    • 继承web服务器权限，执行系统命令
    • 继承web服务器权限，读写文件
    • 反弹shell
    • 控制网站和服务器

02 命令执行的相关函数

测试前，首先要检查以下函数是否在php.ini中被’disable_functions=’禁止

  1. system()
    将字符串作为系统命令执行，并且输出结果

执行命令成功则返回true，否则返回false
测试代码如下：

<?php
if(isset($_GET['cmd'])){
    echo '<pre>';
    system($_GET['cmd']);
}else{
    echo 'no cmd input !';
}
?>

GET方式传入参数cmd=whoami，成功执行命令

  1. shell_exec()
    将字符串作为系统命令执行，不输出结果
    测试代码如下：

    <?php
if(isset($_GET['cmd'])){
    print shell_exec($_GET['cmd']);
}else{
    echo 'no cmd input !';
}
?>

    结果与system()，但没有使用<pre>标签，格式有所不同

  2. exec()
    将字符串作为系统命令执行，返回结果最后一行
    如果添加第二个参数$array，就将每一行结果存入数组之中
    测试代码如下：

    <?php
if(isset($_GET['cmd'])){
    exec($_GET['cmd'],$array);
    print_r($array);
}else{
    echo 'no cmd input !';
}
?>

    执行whoami命令结果如下

  1. passthru()
    与system()类似，将字符串作为系统命令执行，并且输出结果
    无返回值，可以输出二进制数据
    测试代码如下：

    <?php
if(isset($_GET['cmd'])){
    passthru($_GET['cmd']);
}else{
    echo 'no cmd input !';
}
?>

  2. popen()
    执行命令不返回命令结果，而是返回一个文件指针（需要有写权限）
    测试代码如下：

    <?php
if(isset($_GET['cmd'])){
    $cmd=$_GET['cmd'].">> 1.txt";
    popen($cmd,r);
}else{
    echo 'no cmd input !';
}
?>

    执行whoami命令之后，生成了一个1.TXT文件，内容为命令结果

  3. 反引号
    反引号``内的字符串会被当做系统命令执行

    <?php
if(isset($_GET['cmd'])){
    $cmd=$_GET['cmd'];
  	print `$cmd`;
}else{
    echo 'no cmd input !';
}
?>

    同样可以执行系统命令

04 拼接符介绍

  • windows下的拼接符：
    • & 前面语句为假则直接执行后边的
    • && 前面的语句为假就终止执行
    • | 直接执行后边的语句
    • || 前面出错执行后边的
  • linux下的拼接符：
    • ；前面执行完执行后边的
    • & 无论前边的语句真假都会执行
    • && 只有前面语句为真，才会执行后边语句
    • |显示后边的执行结果
    • || 前面出错执行后边的

05 漏洞防范

  1. 减少相关函数的使用，并在disable_functions中禁用函数
  2. 执行命令前对参数进行过滤如escapeshellcmd(),escapeshellarg()
  3. 参数的值使用引号包裹，并使用addslashes函数转义
  4. 对参数设置白名单
