文件下载（文件读取）、文件删除

PHP文件操作漏洞

01 文件下载（文件读取）

  1. 原理
    文件下载与文件读取类似，原因在于程序在下载或者读取文件时，参数filename是用户可控的，使得攻击者可以读取敏感文件

  2. 相关函数

    • file_get_contents()
    • highlight_file()
    • fopen()
    • readfile()
    • fread()
    • fgetss()
    • fgets()
    • parse_ini_file()
    • show_source()
    • file()

    除此之外，文件包含的函数include等也可以通过PHP输入输出流来读取文件 php://filter/

  3. phpcms v9的文件读取漏洞

    // 路径 /phpcms/modules/search/index.php
<?php
  public function public_get_suggest_keyword(){
  $url=$_GET['url'].'&q='.$_GET['q'];
  $res=@file_get_contents($url);
  if(CHARSET != 'gbk'){
    $res=iconv('gbk',CHARSET,$res);
  }
  echo $res
}
?>

    可以看到代码中读取的文件的路径URL是使用用户传入的参数拼接而成的

    如果我们提交url为空，q为1.php，拼接的路径为url=&q=1.php

    要想读取到1.php，需要将&q去掉，可以将其看做目录，使用../进行跳出

    即传入q=../../1.php,此时url=&q=../../1.php

    此时就将&q=当成了目录进行跳过，可以进行文件读取

02 文件删除

  1. 简介
    与文件读取类似，使用户可以删除敏感文件
  2. 相关函数
    • unlink()
    • 老版本 session_destroy()
  3. 利用方法
    删除当前目录下的文件或者使用../跳转来删除其他目录下的文件

03 文件操作漏洞的防范

  1. 文件操作漏洞的共同点
    • 由越权操作使用户可以操作未授权操作的文件
    • 通过跳转目录的方法来操作更多文件
    • 在请求中传入文件名或路径
  2. 防御手段
    • 对权限的管理要合理
    • 检查参数中是否有../等跳转符号
    • 不直接传入文件名，可以使用MD5加密等方式
